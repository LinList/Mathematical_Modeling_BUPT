# 本节导读

> 三天三夜，三个人，一篇论文。怎么分工？怎么安排时间？这个文件给你一份可执行的操作手册。

---

## 一、团队角色详解

### 🧠 建模成员

**你是团队的"大脑"**，负责把现实问题转化为数学模型。

**日常工作：**
- 读题，分析问题的数学本质
- 选择合适的模型（评价？优化？预测？）
- 推导公式，建立模型
- 和编程沟通：模型怎么算、需要什么数据
- 和论文沟通：模型的逻辑怎么讲清楚

**需要储备：**
- 熟悉各类模型（评价类、优化类、预测类、微分方程、图论）
- 知道每个模型的适用场景和优缺点
- 能快速判断"这题该用什么模型"

**学习资料** → 见 `01_建模成员/` 文件夹

---

### 💻 编程成员

**你是团队的"执行者"**，负责把模型变成代码、跑出结果。

**日常工作：**
- 搭建编程环境
- 数据读取、清洗、预处理
- 实现建模成员设计的算法
- 生成结果表格和可视化图表
- 调试代码、处理报错
- 把结果导出给论文成员使用

**需要储备：**
- Python（主力）+ 常用库（numpy, pandas, scipy, sklearn, matplotlib）
- 能快速实现常见模型算法
- 数据处理和可视化能力
- MATLAB（可选，某些题目更方便）

**学习资料** → 见 `02_编程成员/` 文件夹

---

### 📝 论文成员

**你是团队的"表达者"**，负责把团队的工作写成一篇专业论文。

**日常工作：**
- 搭建论文框架和提纲
- 撰写问题重述、问题分析
- 跟进建模进度，把模型写成文字
- 收集编程的结果，制作图表
- 撰写摘要（最重要！）
- 排版、检查、终稿

**需要储备：**
- 论文结构和写作规范
- 公式编辑能力（MathType / LaTeX）
- 图表制作和排版技能
- 摘要写作技巧（30秒抓住评委）

**学习资料** → 见 `03_论文成员/` 文件夹

## 二、三天时间规划（黄金版）

### 比赛时间：周五 18:00 ~ 周一 20:00（共约74小时）

### 📅 Day 1（周五晚 ~ 周六）—— 读题与建模

| 时间 | 建模 | 编程 | 论文 |
|------|------|------|------|
| 周五 18:00-20:00 | 3人一起读题，讨论选题 | 同左 | 同左 |
| 周五 20:00-22:00 | 分析题目，查文献，确定建模思路 | 搭环境，找数据，预处理 | 列论文提纲，写问题重述初稿 |
| 周五 22:00-24:00 | 确定第一个子问题的模型 | 开始写第一部分代码 | 写问题分析、模型假设 |

**Day 1 目标**：选定题目，确定整体建模思路，完成第一问的模型建立。

> 💡 **选题决策**：A题偏物理/连续，B题偏离散/运筹，C题偏大数据。选你们最擅长的方向！讨论时间不要超过2小时，果断决定。

|------|------|------|------|

## 三、团队协作机制

### 1. 同步会制度

- **频率**：每天至少3次同步会（早、中、晚各一次，每次15-20分钟）
- **内容**：每人说三句话——"我完成了什么/我正在做什么/我遇到了什么问题"
- **目的**：确保信息对齐，及时发现问题

### 2. 文件共享

建议用以下方式共享文件：

```
团队共享文件夹/
├── 题目/          → 原始题目和数据
├── 代码/          → 编程成员的代码和结果
├── 论文/          → 论文成员的文档
├── 图表/          → 编程生成的图片
└── 参考/          → 查到的文献资料
```

### 3. 信息流转

```
建模：模型方案 + 公式
        ↓ 传达给
编程：代码实现 + 结果数据 + 图表
        ↓ 传达给
论文：论文正文 + 图表嵌入
        ↑ 反馈给
建模/编程：论文哪里不清楚，补充解释
```

### 4. 应急预案

| 情况 | 应对 |
|------|------|
| 模型太难，编不出来 | 简化模型！先出结果再优化 |
| 数据有问题 | 论文里写明数据局限性，做敏感性分析 |
| 编程报错搞不定 | 上网搜、问队友、换语言（Python↔MATLAB） |
| 论文写不完 | 摘要和模型建立优先，其他从简 |
| 团队意见分歧 | 听建模的（模型方向）+ 听论文的（表达方向）|

## 五、心态管理

- **不要追求完美**：数学建模论文没有"标准答案"，评委看的是你的思路和方法是否合理
- **时间管理 > 模型华丽**：一个简单但完整跑通的模型，比一个复杂但没出结果的模型强100倍
- **保持沟通**：三个人闷头各干各的是最大的敌人
- **注意休息**：三天确实累，但别通宵到崩溃。Day 1 和 Day 2 适当休息，Day 3 才冲刺
- **享受过程**：不管结果如何，这三天的经历本身就很宝贵

祝你们顺利！🏆
