## 二、评价类模型

> **什么时候用**：需要在多个方案/对象中选出"最好"的，或对多个对象进行综合排名打分。

| 模型 | 适用场景 | 优点 | 缺点 | 难度 |
|------|----------|------|------|------|
| **层次分析法（AHP）** | 多准则决策，主观+客观结合，方案选择 | 简单直观，适合定性+定量综合决策 | 主观性强，判断矩阵可能不一致 | ⭐⭐ |
| **TOPSIS法** | 多指标综合评价，有明确数据的多方案排序 | 计算简单，充分利用原始数据，结果直观 | 对数据量纲敏感，需先标准化 | ⭐⭐ |
| **模糊综合评价** | 评价指标难以精确量化（如"好""较好""一般"） | 能处理模糊信息，适合定性评价 | 隶属度函数确定主观性强 | ⭐⭐⭐ |
| **灰色关联分析** | 数据样本少，分析各因素与参考序列的关联程度 | 适合小样本数据，计算简单 | 结果受参考序列选择影响 | ⭐⭐ |
| **熵权法** | 客观赋权，基于数据离散程度确定权重 | 客观赋权，避免主观偏差 | 只能反映数据离散性，不能反映重要性 | ⭐⭐ |

### 评价类模型选择建议

```
有数据吗？
├── 有客观数据 → 需要客观赋权吗？
│   ├── 是 → 熵权法 + TOPSIS
│   └── 否 → TOPSIS（主观赋权）
└── 数据少/偏定性 → 
    ├── 有明确层次结构 → 层次分析法（AHP）
    ├── 指标有模糊性 → 模糊综合评价
    └── 分析因素关联度 → 灰色关联分析
```

> 💡 **小贴士**：实际比赛中，经常组合使用。例如用"AHP确定主观权重 + 熵权法确定客观权重 + TOPSIS排序"，这种"组合拳"往往更受评委青睐。

## 三、优化类模型

> **什么时候用**：需要在满足一定约束条件下，使某个目标达到最优（最大或最小）。

| 模型 | 适用场景 | 优点 | 缺点 | 难度 |
|------|----------|------|------|------|
| **线性规划（LP）** | 目标函数和约束都是线性的，资源分配、生产计划 | 理论成熟，求解高效（单纯形法） | 只能处理线性关系 | ⭐⭐ |
| **非线性规划（NLP）** | 目标或约束含非线性项，投资组合优化 | 适用范围广 | 求解困难，可能陷入局部最优 | ⭐⭐⭐ |
| **整数规划（IP）** | 变量必须取整数，选址、排班 | 能处理离散决策 | 求解复杂度高于线性规划 | ⭐⭐⭐ |
| **0-1规划** | 变量只能取0或1，背包问题、选址问题 | 模型简洁，适用面广 | 本质是NP难问题 | ⭐⭐⭐ |
| **目标规划** | 多目标决策，各目标有优先级 | 能处理冲突的多目标 | 目标优先级设定主观 | ⭐⭐⭐ |
| **动态规划（DP）** | 多阶段决策问题，最短路径、资源分配 | 适合分阶段决策，原理优美 | 状态定义困难，维度灾难 | ⭐⭐⭐⭐ |
| **遗传算法（GA）** | 复杂优化问题，组合优化 | 全局搜索能力强，适用面广 | 参数调优复杂，收敛慢 | ⭐⭐⭐⭐ |
| **模拟退火（SA）** | 组合优化，TSP等 | 能跳出局部最优 | 收敛速度较慢 | ⭐⭐⭐⭐ |

### 优化类模型选择建议

```
目标函数和约束都是线性的吗？
├── 是 → 变量需要取整数吗？
│   ├── 否 → 线性规划（LP）⭐推荐
│   └── 是 → 整数规划 / 0-1规划
└── 否 → 
    ├── 能分阶段处理吗？ → 动态规划
    ├── 问题非常复杂/规模大？ → 遗传算法/模拟退火
    └── 一般非线性 → 非线性规划
```

> 💡 **小贴士**：比赛中遇到优化问题，第一步先把模型写成标准形式（明确目标函数、决策变量、约束条件），然后再决定用什么方法求解。**"先建模型，再选算法"**是铁律。

## 四、预测类模型

> **什么时候用**：需要根据历史数据预测未来趋势。

| 模型 | 适用场景 | 优点 | 缺点 | 难度 |
|------|----------|------|------|------|
| **灰色预测GM(1,1)** | 数据量少（≥4个），短期预测 | 需要数据少，计算简单 | 只适合短期、近似指数增长的数据 | ⭐⭐ |
| **一元/多元回归分析** | 变量间存在因果关系，数据量较多 | 理论成熟，解释性强 | 需满足线性假设和统计假设 | ⭐⭐ |
| **时间序列（ARIMA）** | 有大量历史数据，数据有时间依赖性 | 适合捕捉时间趋势和季节性 | 需要较多数据，模型识别复杂 | ⭐⭐⭐ |
| **移动平均/指数平滑** | 短期预测，数据波动较平稳 | 计算极其简单 | 无法捕捉复杂趋势 | ⭐ |
| **BP神经网络** | 复杂非线性关系，输入输出关系不明确 | 强大的非线性拟合能力 | 需要大量数据，"黑箱"不可解释 | ⭐⭐⭐⭐ |
| **支持向量机（SVR）** | 小样本非线性回归预测 | 小样本表现好，泛化能力强 | 核函数和参数选择复杂 | ⭐⭐⭐⭐ |

### 预测类模型选择建议

```
数据量多少？
├── 很少（4~10个）→ 灰色预测GM(1,1)
├── 中等（10~50个）→ 
│   ├── 有明确因果变量？ → 回归分析
│   └── 无明确因果，仅时间序列？ → 指数平滑
└── 较多（>50个）→
    ├── 关系复杂/非线性？ → BP神经网络 / SVR
    └── 有时间趋势和季节性？ → ARIMA时间序列
```

> 💡 **小贴士**：预测模型的选择很大程度上取决于数据量。比赛时数据往往很少，所以**灰色预测**是最高频出场的模型之一，务必掌握。同时，用多个模型预测并对比结果，是一种很好的加分策略。

## 五、分类/判别类模型

> **什么时候用**：需要把对象分到不同类别，或判断某个对象属于哪一类。

| 模型 | 适用场景 | 优点 | 缺点 | 难度 |
|------|----------|------|------|------|
| **Logistic回归** | 二分类/多分类，判断概率 | 简单成熟，输出概率值，解释性好 | 只能处理线性决策边界 | ⭐⭐ |
| **判别分析（Fisher/LDA）** | 已知类别，对新样本分类 | 理论成熟，计算简单 | 需满足正态分布假设 | ⭐⭐⭐ |
| **聚类分析（K-means/层次聚类）** | 无标签数据分组，市场细分 | 无需标签，自动发现数据结构 | 需预先指定簇数（K-means） | ⭐⭐ |
| **支持向量机（SVM）** | 小样本分类，非线性分类 | 泛化能力强，适合高维数据 | 核函数选择有难度 | ⭐⭐⭐⭐ |

### 分类模型选择建议

```
有标签数据吗？
├── 有 → 
│   ├── 二分类且需要概率 → Logistic回归
│   ├── 多分类，数据服从正态 → 判别分析（LDA）
│   └── 非线性分类，样本少 → SVM
└── 无 → 聚类分析（K-means / 层次聚类）
```

## 六、微分方程类模型

> **什么时候用**：描述随时间连续变化的动态过程，如人口增长、传染病传播、化学反应等。

| 模型 | 适用场景 | 优点 | 缺点 | 难度 |
|------|----------|------|------|------|
| **Malthus人口模型** | 短期人口/细菌增长预测 | 模型极简，揭示指数增长规律 | 长期不准（增长无上限） | ⭐ |
| **Logistic阻滞增长** | 资源有限环境下的增长 | 能描述S型增长曲线，符合实际 | 参数估计需要数据 | ⭐⭐ |
| **Lotka-Volterra捕食模型** | 两个物种互相影响的动态 | 揭示捕食关系的周期性振荡 | 假设较理想化 | ⭐⭐⭐ |
| **SIR传染病模型** | 传染病传播预测 | 能描述疫情传播规律 | 未考虑潜伏期、出生死亡等 | ⭐⭐⭐ |
| **SEIR模型** | 含潜伏期的传染病模型 | 比SIR更贴合实际 | 参数更多，估计更复杂 | ⭐⭐⭐⭐ |

### 微分方程模型选择建议

```
要描述什么过程？
├── 单种群体增长 →
│   ├── 无资源限制 → Malthus模型
│   └── 有环境容量 → Logistic模型
├── 两种群相互作用 →
│   ├── 捕食关系 → Lotka-Volterra模型
│   └── 竞争关系 → 竞争模型
└── 传染病传播 →
    ├── 无潜伏期 → SIR模型
    └── 有潜伏期 → SEIR模型
```

## 七、图论与网络类模型

> **什么时候用**：涉及网络结构、路径、连通性等问题，如交通网络、通信网络、物流配送等。

| 模型/算法 | 适用场景 | 优点 | 缺点 | 难度 |
|-----------|----------|------|------|------|
| **最短路径（Dijkstra）** | 求两点间最短路径 | 算法经典，效率高 | 仅适用于非负权边 | ⭐⭐ |
| **最小生成树（Kruskal/Prim）** | 网络连通最小成本 | 算法简洁高效 | 只解决连通性问题 | ⭐⭐ |
| **网络最大流** | 网络流量分配，管道运输 | 理论成熟 | 仅考虑流量上限 | ⭐⭐⭐ |
| **旅行商问题（TSP）** | 经过所有点的最短回路 | 经典组合优化问题 | NP难问题，精确解困难 | ⭐⭐⭐⭐ |
| **最小费用最大流** | 兼顾费用和流量的网络优化 | 更贴近实际 | 模型和求解更复杂 | ⭐⭐⭐ |

### 图论模型选择建议

```
问题类型是什么？
├── 求最短路径 → Dijkstra算法 / Floyd算法
├── 网络连通最小成本 → 最小生成树（Kruskal/Prim）
├── 流量分配 → 最大流 / 最小费用最大流
├── 经过所有点的回路 → TSP（需用启发式算法）
└── 匹配问题 → 匈牙利算法
```

## 八、"选题→选模型"决策流程

下面是一个完整的决策流程，帮你从赛题出发，一步步确定用什么模型：

### 第一步：判断问题类型

```
拿到赛题后，问自己：题目要干什么？
├── "评价/比较/排序" → 评价类问题
├── "最优/最大/最小/最多/最少" → 优化类问题
├── "预测/估计未来" → 预测类问题
├── "分类/分组/判断属于哪类" → 分类判别类问题
├── "描述变化过程/随时间演变" → 微分方程类问题
└── "路径/网络/连通/分配" → 图论网络类问题
```

### 第二步：分析数据特征

```
数据多还是少？
├── 数据很少（<10个）→ 灰色预测、灰色关联、AHP
├── 数据中等 → 回归、TOPSIS、熵权法
└── 数据较多 → 神经网络、时间序列、聚类分析
```

### 第三步：确定模型

综合问题类型和数据特征，选择1-2个主模型，再考虑是否需要组合。

### 第四步：考虑组合模型

实际比赛中，单一模型往往不够，常用的组合策略：

| 组合方式 | 典型场景 |
|----------|----------|
| AHP + TOPSIS | AHP确定权重，TOPSIS排序 |
| 熵权法 + TOPSIS | 熵权法客观赋权，TOPSIS排序 |
| 灰色预测 + 回归 | 灰色预测短期趋势，回归分析因果关系 |
| 主成分分析 + 聚类 | 降维后聚类 |
| 微分方程 + 优化 | 先建微分方程模型，再优化参数 |

### 完整决策流程图（文字版）

```
【1】理解题目
  │
  ▼
【2】判断问题类型（评价/优化/预测/分类/微分方程/图论）
  │
  ▼
【3】分析数据特征（数据量、数据类型、是否有时间序列）
  │
  ▼
【4】查阅文献，看前人怎么解决类似问题
  │
  ▼
【5】初步选定1~2个模型
  │
  ▼
【6】考虑模型组合策略（是否需要组合多个模型？）
  │
  ▼
【7】建立模型，求解，检验
  │
  ▼
【8】模型效果不好？→ 回到【5】换模型或调整
  │
  ▼
【9】模型效果良好 → 撰写论文
```

## 九、各模型的"出场频率"参考

根据历年CUMCM和MCM/ICM赛题统计，以下模型出场频率较高：

| 频率等级 | 模型 |
|----------|------|
| ⭐⭐⭐⭐⭐ 极高频 | 层次分析法、TOPSIS、灰色预测、线性规划、回归分析 |
| ⭐⭐⭐⭐ 高频 | 熵权法、聚类分析、微分方程模型、最短路径 |
| ⭐⭐⭐ 中频 | 模糊综合评价、整数规划、动态规划、时间序列 |
| ⭐⭐ 低中频 | BP神经网络、遗传算法、模拟退火、SVM |
| ⭐ 偶尔出现 | 偏微分方程、复杂网络分析、博弈论 |

> 🎯 **学习建议**：优先掌握"极高频"和"高频"模型，这些是比赛中大概率会用到的。后续章节我们将逐一详细讲解这些模型的原理和实例。

## 十、小结

| 要点 | 内容 |
|------|------|
| 六大类模型 | 评价类、优化类、预测类、分类判别类、微分方程类、图论网络类 |
| 选模型三步走 | 判断问题类型→分析数据特征→确定模型 |
| 组合策略 | 多模型组合往往比单一模型效果更好 |
| 优先学习 | AHP、TOPSIS、灰色预测、线性规划、回归分析（出场率最高） |

> 💡 **小贴士**：不要试图一次学完所有模型！先掌握最常用的5-6个核心模型，能灵活运用就已经能应对大部分赛题了。后续章节会带你逐一攻破这些核心模型，每个都配有完整实例。
