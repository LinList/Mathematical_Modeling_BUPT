## 一、Python 环境搭建

### 1.1 为什么选 Python？

在数学建模竞赛中，Python 是目前最流行的编程语言，原因如下：

| 优点 | 说明 |
|------|------|
| 语法简洁 | 比 C 语言简单得多，写起来快 |
| 生态丰富 | 有大量科学计算库，直接调用 |
| 社区活跃 | 遇到问题搜一下基本都能找到答案 |
| 免费开源 | 不需要花钱买许可证 |

> **小贴士**：如果你之前学过 C 语言，转 Python 会非常快。Python 不需要写分号，不需要声明变量类型，也不需要手动管理内存。

### 1.2 安装 Anaconda

Anaconda 是一个集成了 Python + 常用科学计算库的"全家桶"。安装它，你就不用一个个去装库了。

**安装步骤：**

1. 打开浏览器，访问 Anaconda 官网：https://www.anaconda.com/download
2. 下载对应操作系统的安装包（Windows / macOS / Linux）
3. 双击安装，一路"Next"，建议勾选"Add Anaconda to my PATH"（虽然安装时会提示不推荐，但对于初学者来说方便很多）

**验证安装是否成功：**

打开命令行（Windows 按 `Win+R`，输入 `cmd`），输入：

```bash
conda --version
```

如果显示类似 `conda 23.x.x`，说明安装成功。

### 1.3 创建虚拟环境

虚拟环境的作用是隔离不同项目的依赖。比如你打比赛时用的库版本和平时练习时可能不同，用虚拟环境就不会冲突。

```bash
# 创建一个名为 modeling 的虚拟环境，Python 版本为 3.10
conda create -n modeling python=3.10

# 激活环境
conda activate modeling

# 退出环境（不需要时用）
conda deactivate
```

> **避坑指南**：每次打开命令行写代码前，记得先 `conda activate modeling` 激活环境！否则你安装的库可能找不到。

### 1.4 安装库的两种方式

```bash
# 方式一：用 conda 安装（推荐，兼容性好）
conda install numpy pandas scipy matplotlib

# 方式二：用 pip 安装（库更全，更新更快）
pip install numpy pandas scipy matplotlib
```

> **小贴士**：conda 和 pip 可以混用，但建议优先用 conda。如果 conda 找不到某个库，再用 pip。

## 二、数学建模必装库一览表

以下是数学建模中最常用的 Python 库，建议一次性全部安装。

| 库名 | 用途 | 安装命令 |
|------|------|----------|
| **numpy** | 科学计算基础库，提供高效的多维数组和矩阵运算 | `pip install numpy` |
| **pandas** | 数据处理与分析利器，类似 Excel 但强大百倍 | `pip install pandas` |
| **scipy** | 科学计算工具箱，包含优化、积分、插值、统计等模块 | `pip install scipy` |
| **matplotlib** | 最基础的绘图库，能画各种图表 | `pip install matplotlib` |
| **seaborn** | 基于 matplotlib 的统计绘图库，画图更好看 | `pip install seaborn` |
| **scikit-learn** | 机器学习库，含回归、分类、聚类等算法 | `pip install scikit-learn` |
| **networkx** | 复杂网络/图论计算库，用于网络分析 | `pip install networkx` |
| **sympy** | 符号计算库，能做代数运算、求导、解方程 | `pip install sympy` |
| **pulp** | 线性规划/整数规划求解器 | `pip install pulp` |
| **scikit-opt** | 启发式算法库（遗传算法、粒子群等） | `pip install scikit-opt` |
| **openpyxl** | 读写 Excel 文件 | `pip install openpyxl` |

**一键安装命令：**

```bash
pip install numpy pandas scipy matplotlib seaborn scikit-learn networkx sympy pulp scikit-opt openpyxl
```

> **避坑指南**：如果安装速度慢，可以使用国内镜像源：
> ```bash
> pip install numpy -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```
> 或者永久设置清华镜像：
> ```bash
> pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
> ```

## 三、MATLAB 简介

如果你的学校有 MATLAB 许可证，也可以使用 MATLAB。MATLAB 在数学建模中有以下优势：

| 优点 | 缺点 |
|------|------|
| 矩阵运算极其方便 | 收费软件，价格昂贵 |
| 自带大量工具箱 | 语法不如 Python 通用 |
| 画图质量高 | 社区不如 Python 活跃 |
| Simulink 仿真能力强 | |

### MATLAB 基础语法速览

```matlab
% 注释用百分号

% 定义矩阵
A = [1 2 3; 4 5 6; 7 8 9];

% 矩阵运算
B = A';          % 转置
C = A * B;       % 矩阵乘法
D = A.^2;        % 逐元素平方（注意有个点）

% 解线性方程组 Ax = b
b = [1; 2; 3];
x = A \ b;       % 左除，等价于 inv(A)*b

% 画图
x = 0:0.1:2*pi;
y = sin(x);
plot(x, y);
title('正弦曲线');
xlabel('x');
ylabel('sin(x)');

% 线性规划
% min f'x  s.t. Ax <= b, Aeq*x = beq, lb <= x <= ub
f = [-5; -4];
A = [6 4; 1 2; -1 1];
b = [24; 6; 1];
lb = [0; 0];
[x, fval] = linprog(f, A, b, [], [], lb);
```

> **小贴士**：在数学建模比赛中，Python 和 MATLAB 选一个主力使用即可。推荐 Python，因为免费且通用。但如果队伍中有人擅长 MATLAB，也可以混用。

## 四、Jupyter Notebook 使用指南

Jupyter Notebook 是数学建模编程的"神器"。它可以在浏览器中运行代码，且代码、输出、文字说明混排，非常适合探索性分析和写报告。

### 4.1 启动 Jupyter Notebook

```bash
# 激活环境后，在命令行输入
conda activate modeling
jupyter notebook
```

浏览器会自动打开 Jupyter Notebook 页面。

### 4.2 基本操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 运行当前单元格 | `Shift + Enter` | 运行代码并跳到下一个单元格 |
| 运行不跳转 | `Ctrl + Enter` | 运行代码但停留在当前单元格 |
| 新建单元格 | 点击 `+` 按钮 | 在当前单元格下方新建 |
| 切换为代码 | `Esc` 然后 `Y` | 当前单元格变为代码类型 |
| 切换为 Markdown | `Esc` 然后 `M` | 当前单元格变为文字说明 |
| 删除单元格 | `Esc` 然后 `DD` | 连按两次 D 删除 |
| 保存 | `Ctrl + S` | 保存文件 |

### 4.3 Jupyter 中的魔法命令

```python
# 查看所有魔法命令
%lsmagic

# 内联显示 matplotlib 图表（必须加这行）
%matplotlib inline

# 查看变量列表
%who

# 查看变量详情
%whos

# 计算代码运行时间
%timeit [x**2 for x in range(1000)]

# 运行外部 Python 文件
%run my_script.py
```

> **小贴士**：每次新建 Notebook 时，第一件事就是在第一个单元格写 `%matplotlib inline`，这样图表就能直接显示在 Notebook 里。

### 4.4 推荐安装 Jupyter 扩展

```bash
# 安装 nbextensions（提供代码折叠、目录等功能）
conda install -c conda-forge jupyter_nbextensions_configurator
jupyter nbextensions_configurator enable --user
```

安装后在 Notebook 页面会多出一个 "Nbextensions" 标签，推荐开启以下扩展：
- **Table of Contents (2)**：自动生成目录
- **Hinterland**：代码自动补全
- **Codefolding**：代码折叠

## 五、VS Code / PyCharm 推荐与配置

### 5.1 VS Code（推荐）

VS Code 是微软出品的免费编辑器，轻量且强大。

**安装步骤：**

1. 访问 https://code.visualstudio.com/ 下载安装
2. 安装 Python 扩展：打开 VS Code，按 `Ctrl+Shift+X` 打开扩展商店，搜索 "Python"，安装第一个（微软官方出品）

**推荐安装的扩展：**

| 扩展名 | 用途 |
|--------|------|
| Python | Python 语言支持（必装） |
| Jupyter | 在 VS Code 中运行 Jupyter Notebook |
| Pylance | Python 智能提示 |
| Code Runner | 一键运行代码 |
| Chinese Language Pack | 中文界面 |
| GitLens | Git 增强 |

**配置 Python 解释器：**

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 `Python: Select Interpreter`
3. 选择你创建的 `modeling` 虚拟环境

> **小贴士**：VS Code 可以直接打开 `.ipynb` 文件运行 Jupyter Notebook，不需要打开浏览器！

### 5.2 PyCharm

PyCharm 是 JetBrains 出品的 Python IDE，功能强大但较重。

- **Community 版**：免费，功能足够数学建模使用
- **Professional 版**：收费，支持科学模式、数据库等

**安装步骤：**

1. 访问 https://www.jetbrains.com/pycharm/download/ 下载 Community 版
2. 安装时勾选 "Add to PATH"

**配置虚拟环境：**

1. `File` → `Settings` → `Project: xxx` → `Python Interpreter`
2. 点击齿轮图标 → `Add` → `Conda Environment` → 选择 `modeling`

> **小贴士**：对于初学者，推荐 VS Code，因为它更轻量、启动更快。如果喜欢功能全面的 IDE，选 PyCharm。

## 六、编程成员的职责清单

在数学建模团队中，编程成员扮演着"翻译官"的角色——把建模成员的数学公式翻译成代码，把数据变成图表和结论。

### 6.1 比赛前

- [ ] 搭建好编程环境（按本节操作）
- [ ] 熟练掌握 NumPy / Pandas 基本操作
- [ ] 提前准备好常用算法的代码模板（评价类、优化类、预测类）
- [ ] 准备好数据可视化模板（折线图、柱状图、散点图、热力图等）
- [ ] 和建模成员确认他们可能用到的模型，提前写好代码框架
- [ ] 和论文成员确认图表导出格式和分辨率要求

### 6.2 比赛中

- [ ] **第一天**：和团队一起理解题目，参与讨论建模思路，准备好数据读取和预处理代码
- [ ] **第二天**：根据建模成员的方案，实现核心模型代码，开始画图
- [ ] **第三天**：优化代码，跑结果，画最终配图，把结果导出给论文成员
- [ ] 全程保持代码可运行，每完成一个功能就测试一次
- [ ] 代码要写注释！方便论文成员理解

### 6.3 比赛后

- [ ] 整理代码，保留可复用的部分
- [ ] 总结本次比赛的经验教训
- [ ] 更新自己的代码模板库

### 6.4 编程成员的核心能力要求

```
┌─────────────────────────────────────────────┐
│           编程成员核心能力地图               │
├──────────────┬──────────────────────────────┤
│  数据处理     │ 读取CSV/Excel、清洗、变换    │
│  算法实现     │ 评价/优化/预测模型编码        │
│  数据可视化   │ 画出论文级别的高质量图表      │
│  结果输出     │ 导出表格、图片供论文使用      │
│  调试排错     │ 快速定位和修复代码错误        │
└──────────────┴──────────────────────────────┘
```

> **避坑指南**：比赛中最容易犯的错误就是"等到最后才跑代码"。一定要边写边测，确保每一步都能正常运行。不要等到第三天晚上才发现代码跑不通！
