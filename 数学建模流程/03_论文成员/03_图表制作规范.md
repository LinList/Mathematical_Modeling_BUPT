# 图表制作规范

## 本节导读

> 一图胜千言。在数学建模论文里，一张好的图表能让评委瞬间理解你的结果，而一张差的图表能让评委瞬间失去耐心。
>
> 本节讲清楚：什么情况用什么图、图表怎么编号怎么引用、图表标题怎么写、怎么做出竞赛级的图表、好图表和差图表到底差在哪里。读完本节，你的图表水平能超过80%的参赛队伍。

---

## 一、图表在论文中的重要性

### 为什么图表这么重要

1. **评委看图比看文字快**：一堆数字和一段文字，评委更愿意看图
2. **图表最能展示结果**：趋势、对比、分布，用图最直观
3. **图表体现专业度**：粗糙的图表 = 不认真 = 低分；精美的图表 = 专业 = 加分
4. **图表帮助说理**：很多时候，一张图就能证明"结果合理"，比写三段话都管用

### 论文中图表的分布

一篇典型的竞赛论文，图表数量大概在 **8-15个** 之间（含表格）。分布大致是：

| 论文部分 | 图表数量 | 类型 |
|---------|---------|------|
| 问题分析 | 1-2个 | 思路流程图 |
| 模型建立 | 2-3个 | 模型框架图、算法流程图 |
| 模型求解/结果 | 4-6个 | 数据图、结果对比图 |
| 模型检验 | 2-3个 | 敏感性分析图、误差图 |
| 符号说明/数据 | 1-2个 | 三线表 |

> **小贴士**：图表不是越多越好。每个图表都要有存在的理由——它必须帮助说明某个观点或展示某个结果。如果你发现一个图表删掉也不影响论文理解，那就删掉。

## 二、图表类型选择指南

### 什么情况用什么图

#### 1. 流程图：模型思路图怎么画

**适用场景**：展示建模思路、算法流程、整体框架。

**什么时候用**：
- 问题分析部分，画一个整体思路图
- 模型建立部分，画算法流程图
- 多个模型串联时，画框架图

**怎么画好**：
- 用方框+箭头，方框里写步骤名称
- 层次分明：输入→处理→输出
- 颜色不超过3种，保持简洁
- 工具推荐：Visio、ProcessOn、draw.io、PowerPoint

**示例描述**：
> 一个好的流程图：从左到右分为"数据输入→预处理→权重计算→模糊评价→结果输出"五个方框，箭头连接，每个方框下面有一行小字说明具体操作。整体简洁、一目了然。
>
> 一个差的流程图：方框大小不一、箭头乱指、方框里写了一整段话、颜色用了七八种——看起来像一团乱麻。

---

#### 2. 数据图：折线/柱状/散点/热力图的选择

**折线图**：
- 适用：展示时间序列变化、趋势
- 例如：各月份AQI变化趋势、预测值与真实值对比

**柱状图**：
- 适用：展示分类数据的对比、排名
- 例如：各城市空气质量得分对比、各污染物权重对比

**散点图**：
- 适用：展示两个变量之间的关系、相关性
- 例如：PM2.5浓度与AQI的关系散点图

**热力图**：
- 适用：展示矩阵数据、多变量关系
- 例如：各月份各污染物浓度的热力图、相关性矩阵

**饼图**：
- 适用：展示比例构成（少用，信息量低）
- 例如：各污染物权重占比

**箱线图**：
- 适用：展示数据分布特征
- 例如：各季节污染物浓度分布

#### 选择决策树

```
你要展示什么？
├── 趋势变化 → 折线图
├── 分类对比 → 柱状图
├── 变量关系 → 散点图
├── 矩阵数据 → 热力图
├── 比例构成 → 饼图（少用）
├── 数据分布 → 箱线图
└── 流程/思路 → 流程图
```

> **避坑指南**：饼图尽量少用——它只能展示比例，信息量很低，而且分类多了就看不清。如果只是展示权重占比，用柱状图反而更清晰。

---

#### 3. 结果图：拟合图、对比图、灵敏度图

**拟合图**：
- 散点+拟合曲线，展示模型拟合效果
- 例如：预测值vs真实值的散点图，加一条y=x的参考线

**对比图**：
- 柱状图或折线图，展示不同方法/模型的结果对比
- 例如：ARIMA vs LSTM vs 组合模型的预测误差对比

**灵敏度图**：
- 折线图或雷达图，展示参数变化对结果的影响
- 例如：权重变化±20%时评价结果的变化曲线

---

#### 4. 表格：三线表格式规范

**三线表**是数学建模论文的标准表格格式，只有三条横线（顶线、底线、表头线），没有竖线。

**三线表的样子**：

```
┌──────────────────────────────────────┐  ← 顶线（粗线）
│ 月份   PM2.5   PM10   SO₂   AQI等级  │  ← 表头
├──────────────────────────────────────┤  ← 表头线（细线）
│ 1月    85      120    22    III级    │
│ 2月    72      98     18    II级     │
│ ...                                  │
│ 12月   110     156    35    IV级     │
└──────────────────────────────────────┘  ← 底线（粗线）
```

**三线表制作**：
- **Word**：插入表格后，去掉所有边框，只加顶线、表头线、底线（表格设计→边框）
- **LaTeX**：使用 `booktabs` 宏包，`\toprule`、`\midrule`、`\bottomrule`
- **Python**：matplotlib 画表或用 pandas 的 `to_latex`，Style设置

## 三、图表编号与引用规范

### 编号规则

- **图**：按出现顺序编号——图1、图2、图3……
- **表**：按出现顺序编号——表1、表2、表3……
- 图和表**分别编号**，不要混在一起
- 每个图/表在正文中必须被引用

### 正文引用方式

**图的引用**：
> "各月份AQI变化趋势如**图1**所示。"
> "由**图2**可以看出，冬季PM2.5浓度显著高于夏季。"
> "预测结果对比见**图3**，组合模型的拟合效果最优。"

**表的引用**：
> "各指标权重计算结果如**表1**所示。"
> "**表2**列出了各月份的综合评价结果。"

### 引用的注意事项

1. **先引用后出现**：正文中先提到"如图1所示"，图1才在下方出现。不要图先出现、正文后才提到。
2. **每个图表都要被引用**：没有被正文引用的图表，评委不知道你为什么放它。
3. **引用要加粗**：竞赛论文中，"图1""表1"通常加粗显示，方便评委定位。

> **避坑指南**：最常见的问题是"图表没有被引用"或"引用编号对不上"。最后检查时，逐个核对每个图的编号和正文的引用是否一致。

## 四、图表标题怎么写

### 基本规则

- **图题在图下方**，居中
- **表题在表上方**，居中
- 标题要简洁准确，一句话说清楚图表展示的是什么

### 好标题 vs 差标题

**❌ 差标题：**
- 图1：图（什么都没说）
- 图1：结果（什么结果？）
- 图1：各月份的空气质量各项污染物浓度变化趋势对比图（太长太啰嗦）

**✅ 好标题：**
- 图1：2023年各月份AQI变化趋势
- 图2：各污染物指标权重
- 图3：ARIMA与组合模型预测结果对比
- 表1：各指标熵权法计算权重
- 表2：各月份空气质量综合评价结果

### 标题写作技巧

1. **包含"展示什么"和"区分信息"**：如"各月份AQI变化趋势"——展示的是AQI变化趋势，区分信息是"各月份"。
2. **不要以"图"或"表"字结尾**：如"各月份AQI变化趋势图"——"图"字多余，因为编号已经说明是图了。
3. **长度控制在10-20字**：太短说不清，太长啰嗦。

## 五、图表美观度要求

### 分辨率

- 图像分辨率至少 **300 dpi**（用于打印）
- 用Python matplotlib 保存时设 `dpi=300`：
  ```python
  plt.savefig('fig1.png', dpi=300, bbox_inches='tight')
  ```
- 不要截图！截图分辨率低，打印模糊

### 字体大小

- 图中文字字体大小要和正文协调
- 坐标轴标签：10-12pt
- 图例：9-11pt
- 标题：12-14pt
- **正文字号是几号，图里字号不能差太多**

### 颜色选择

- **不要用默认颜色**：matplotlib默认的颜色不太好看
- **推荐配色**：
  - 蓝色系：#2E86AB、#A23B72、#F18F01
  - 学术配色：深蓝、深红、深绿、橙色
  - 用 `seaborn` 库的配色比默认好看很多
- **颜色不超过5种**：一张图里颜色太多显得杂乱
- **考虑黑白打印**：有些评委会黑白打印，用不同线型（实线、虚线、点线）区分，不要只靠颜色

### 其他美观要素

- 坐标轴要有标签和单位（如"月份" "AQI (μg/m³)"）
- 图例不要遮挡数据
- 网格线用浅灰色、细线
- 去掉多余的边框（matplotlib 的 `spines`）

## 六、用Word/Python生成竞赛级图表

### 用Word/Excel画图

**适合**：简单的柱状图、折线图

**步骤**：
1. 在Excel中选中数据，插入图表
2. 双击图表元素进行精细调整：
   - 坐标轴：设置刻度、标签、单位
   - 数据系列：调整颜色、线宽、标记
   - 图例：调整位置和字体
3. 复制图表，粘贴到Word中（选择性粘贴为图片）
4. 在图下方加标题

**Excel画图的美化技巧**：
- 去掉默认的网格线，或改为浅灰色
- 数据标签字体设为10pt
- 颜色手动改为学术配色（右键数据系列→填充颜色）
- 坐标轴标题一定要加

### 用Python画图（推荐）

**适合**：所有类型的图，尤其是复杂数据图

**基础模板**：

```python
import matplotlib.pyplot as plt
import matplotlib
import numpy as np

# 设置中文字体（Windows）
matplotlib.rcParams['font.sans-serif'] = ['SimHei']
matplotlib.rcParams['axes.unicode_minus'] = False

# 设置全局样式
plt.style.use('seaborn-v0_8-whitegrid')

# 创建数据
months = ['1月', '2月', '3月', '4月', '5月', '6月',
          '7月', '8月', '9月', '10月', '11月', '12月']
aqi = [95, 88, 72, 65, 58, 50, 45, 48, 60, 75, 88, 110]

# 画折线图
fig, ax = plt.subplots(figsize=(8, 5))
ax.plot(months, aqi, color='#2E86AB', linewidth=2,
        marker='o', markersize=6, label='AQI')

# 美化
ax.set_xlabel('月份', fontsize=12)
ax.set_ylabel('空气质量指数 (AQI)', fontsize=12)
ax.set_title('2023年各月份AQI变化趋势', fontsize=14)
ax.legend(fontsize=11)

# 设置刻度
ax.tick_params(axis='both', labelsize=10)

# 去掉顶部和右侧边框
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)

# 保存
plt.tight_layout()
plt.savefig('fig1_aqi_trend.png', dpi=300, bbox_inches='tight')
plt.show()
```

**画对比柱状图**：

```python
import matplotlib.pyplot as plt
import numpy as np

models = ['ARIMA', 'LSTM', '组合模型']
mape = [4.3, 3.1, 2.5]
colors = ['#A23B72', '#F18F01', '#2E86AB']

fig, ax = plt.subplots(figsize=(6, 4))
bars = ax.bar(models, mape, color=colors, width=0.5, edgecolor='white')

# 在柱子上方标注数值
for bar, value in zip(bars, mape):
    ax.text(bar.get_x() + bar.get_width()/2, bar.get_height() + 0.1,
            f'{value}%', ha='center', fontsize=11)

ax.set_ylabel('平均绝对百分比误差 (MAPE)', fontsize=12)
ax.set_title('不同模型预测误差对比', fontsize=14)
ax.set_ylim(0, 6)
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)

plt.tight_layout()
plt.savefig('fig2_model_comparison.png', dpi=300, bbox_inches='tight')
plt.show()
```

**画散点拟合图**：

```python
import matplotlib.pyplot as plt
import numpy as np

# 真实值和预测值
y_true = np.array([...])
y_pred = np.array([...])

fig, ax = plt.subplots(figsize=(6, 6))
ax.scatter(y_true, y_pred, color='#2E86AB', alpha=0.6, s=30)

# 画y=x参考线
min_val = min(y_true.min(), y_pred.min())
max_val = max(y_true.max(), y_pred.max())
ax.plot([min_val, max_val], [min_val, max_val],
        'r--', linewidth=1.5, label='y = x (理想拟合)')

ax.set_xlabel('真实值', fontsize=12)
ax.set_ylabel('预测值', fontsize=12)
ax.set_title('预测值与真实值对比', fontsize=14)
ax.legend(fontsize=11)
ax.set_aspect('equal')
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)

plt.tight_layout()
plt.savefig('fig3_scatter_fit.png', dpi=300, bbox_inches='tight')
plt.show()
```

> **小贴士**：Python画图的核心就三步——`fig, ax = plt.subplots()`创建画布、`ax.plot/bar/scatter()`画图、`plt.savefig()`保存。其他都是美化。如果编程成员负责画图，你可以把上面的模板给他参考。

## 七、图表检查清单

写完论文后，逐个检查图表：

| 序号 | 检查项 | 是否做到 |
|------|--------|---------|
| 1 | 每个图/表都有编号（图1、表1） | ☐ |
| 2 | 每个图/表都在正文中被引用 | ☐ |
| 3 | 引用编号和图表编号一致 | ☐ |
| 4 | 图题在图下方，表题在表上方 | ☐ |
| 5 | 标题简洁准确（10-20字） | ☐ |
| 6 | 图表分辨率≥300dpi | ☐ |
| 7 | 字体大小合适，和正文协调 | ☐ |

| 11 | 图表内容与正文描述一致 | ☐ |
| 12 | 没有被遮挡的信息（图例不遮数据） | ☐ |

## 八、好图表 vs 差图表对比

### 对比1：折线图

**好图表**：
> 一张折线图，横轴是1-12月，纵轴是AQI。线条是深蓝色实线，每个数据点有圆形标记。纵轴标签是"AQI (μg/m³)"，横轴标签是"月份"。背景有浅灰色网格线，顶部和右侧无边框。图例在右上角，字号11pt。图下方标题"图1：2023年各月份AQI变化趋势"。整张图干净、清晰、一眼看出AQI先降后升的U型趋势。

**差图表**：
> 一张折线图，用matplotlib默认的紫色线条，没有数据标记。纵轴没有标签，横轴只有数字1-12没有"月份"二字。没有网格线，四周边框都在。图例缺失。标题"图1：图"。分辨率只有72dpi，打印出来是糊的。整张图看不出重点，像草稿。

---

### 对比2：柱状图

**好图表**：
> 一张分组柱状图，对比三种模型（ARIMA、LSTM、组合模型）的预测误差。每个柱子颜色不同（深红、橙色、深蓝），柱子上方标注了具体误差值（4.3%、3.1%、2.5%）。纵轴标签"MAPE (%)"，横轴标签"模型"。标题"图2：不同模型预测误差对比"。柱子宽度适中，间距合理。一眼看出组合模型误差最低。

**差图表**：
> 一张柱状图，三个柱子全是默认的蓝色，分不清哪个是哪个。柱子上方没有标注数值。纵轴刻度从0到100（实际数据都在5以下），柱子矮得几乎看不见。没有纵轴标签。标题"图2：结果"。整张图没有信息量。

---

### 对比3：表格

**好表格**：
> 一张三线表，展示各月份空气质量评价结果。顶线和底线是1.5pt粗线，表头线是0.75pt细线，没有竖线。表头是"月份|PM2.5|PM10|SO₂|综合得分|空气质量等级"，数据排列整齐，数字右对齐、文字居中对齐。标题"表1：各月份空气质量综合评价结果"在表格上方。整张表干净专业。

**差表格**：
> 一张表格，所有边框都有（包括竖线和内部横线），看起来像Excel默认格式。表头和数据没有区分（都是同样字体）。数字有的左对齐有的居中对齐，不统一。有一列数据特别宽，挤出了页面。标题"表1"在表格下方。整张表像草稿纸。

---

### 对比4：散点拟合图

**好图表**：
> 一张散点图，横轴真实值、纵轴预测值，散点是半透明蓝色圆点。图上有一条红色虚线（y=x参考线），散点紧密分布在参考线附近。坐标轴比例1:1（`set_aspect('equal')`）。标题"图3：预测值与真实值对比"。有图例说明"预测值"和"y=x理想拟合"。一眼看出预测精度高（散点贴近参考线）。

**差图表**：
> 一张散点图，用默认的实心蓝点，不透明，数据点重叠时看不清密度。没有参考线。横纵坐标比例不一样，视觉上散点偏离对角线。没有图例。标题"图3：拟合"。整张图无法判断拟合效果好坏。

---

### 对比5：流程图

**好图表**：
> 一张整体思路流程图，从上到下分为四层："数据预处理→权重计算（熵权法）→模糊综合评价→结果输出"。每层是一个圆角矩形，用向下箭头连接。左侧标注"问题一"的流程，右侧标注"问题二"的流程，两个流程在底部汇聚到"综合结论"。颜色用了浅蓝和浅灰两种，简洁清爽。标题"图4：论文整体建模思路"。整张图一眼看懂论文的逻辑框架。

**差图表**：
> 一张流程图，方框大小不一、形状不一（有圆有方有椭圆），箭头方向混乱（有向左的有向右的有向上的）。方框里写了一整段话，字特别小看不清。颜色用了红黄绿蓝紫七种。标题"图4：流程图"。整张图像一团乱麻，评委完全看不懂你想表达什么。

## 本节总结

| 要点 | 内容 |
|------|------|
| 选对图 | 趋势用折线、对比用柱状、关系用散点、矩阵用热力 |
| 三线表 | 论文表格用三线表，不用网格表 |
| 编号引用 | 图表按顺序编号，正文必须引用，图题在下表题在上 |
| 美观度 | 300dpi、字体协调、颜色≤5种、坐标轴有标签 |
| 工具 | 简单图用Excel，专业图用Python matplotlib |
| 好图表 | 干净、清晰、有信息量、一眼看懂 |
| 差图表 | 模糊、杂乱、没标签、没标题、默认配色 |

> **记住一句话**：图表是论文的"颜值担当"。评委翻到你的论文，第一眼看到的就是图表。图表好看，评委觉得你专业；图表难看，评委觉得你敷衍。花点时间把图表做好，是性价比最高的提分方式。
