## 🚀 怎么开始

### 第一步：所有人看总览（30分钟）

打开 `00_总览/` 文件夹，两个文件都看一遍。搞清楚：
- 数学建模是什么
- 比赛怎么打
- 三个人怎么分工
- 三天怎么安排时间

### 第二步：各自学习自己的板块（1-2周）

| 你是 | 看这里 |
|------|--------|
| 🧠 建模 | `01_建模成员/` 全部文件，按顺序学 |
| 💻 编程 | `02_编程成员/` 全部文件，边看边练代码 |
| 📝 论文 | `03_论文成员/` 全部文件，练习写和排版 |

> 💡 建议：也快速浏览一下其他两个板块，了解队友在干什么，方便沟通。

### 第三步：一起做真题模拟（3天）

从 `04_资源/` 里选一套真题，三人按比赛流程完整做一遍。这是最重要的环节！

### 第四步：查漏补缺，准备参赛

根据模拟中发现的问题，回头补学相关内容。检查 `04_资源/` 里的赛前清单。

## 🎯 核心理念

1. **先完成，再完美** —— 简单模型跑出结果 > 复杂模型没结果
2. **沟通 > 个人能力** —— 三个臭皮匠，前提是他们在说话
3. **摘要定生死** —— 评委30秒看摘要，花最多时间打磨它
4. **代码要能跑** —— 复制粘贴就能运行的代码才是好代码
5. **模拟是关键** —— 赛前没模拟过 = 比赛一定翻车
